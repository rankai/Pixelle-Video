# AC-5 数字人口播应用化 Entry（2026-07-20）

状态：`entry_passed_with_boundary`；当前唯一入口已放行至 `APP-IPB/AC-5 implementation`。

## Entry 来源与目标

- 上位入口：`docs/reviews/2026-07-18-application-center-publishing-program-progress.md`。
- 领域要求：应用中心主方案 §“阶段 AC-5：数字人口播应用化”。
- 目标：把已有 `ip_broadcast` 五步口播能力接入统一 ContentProject/AppRun/Artifact/Handoff 体系，复用既有 session、task、LLM、TTS、数字人与媒体渲染内核，不重写生产内核。

## 冻结交付范围

1. `IpBroadcastAppAdapter`：将统一 AppRun 输入映射到已有 session 创建/恢复/步骤执行，输出统一状态与可审计关联。
2. 来源 handoff：支持空白项目、文案 ArtifactVersion、selected_title ArtifactVersion 三种入口；来源版本固定，不读取前端临时状态。
3. 状态映射：冻结 session/task/AppRun 的状态、取消、失败、重试、重启恢复映射；`waiting_user`/`needs_attention` 不得映射为成功。
4. 产物登记：最终视频、封面、发布文案/素材登记为 ArtifactVersion；重复执行幂等，不产生孤儿项目、重复视频或悬挂引用。
5. 兼容迁移：已有 session 可恢复；旧直接口播入口保持可用；新入口由 `digitalHumanInAppCenter` 独立 flag 保护。

## Entry 必须验证的契约

| 契约 | Entry 判定 |
| --- | --- |
| Adapter 所有权 | 只允许修改 `app_center` 口播 adapter、关联 API/UI、契约/fixture/测试；不得重写 `IpBroadcastWorkflow` 核心步骤 |
| 事实源 | AppRun/Artifact 仍以 `app_center.sqlite` 为统一项目事实源；旧 session/task 仍是口播步骤执行事实源；Generic Task 只做投影 |
| 复用模型 | 复用现有 `AppLLMPort`/`ConfigManager/PixelleVideoConfig.llm`；不新增 provider/client/model 配置源 |
| 产物来源 | ArtifactVersion 必须记录 project、source AppRun、版本与文件可信路径；PublishPackage 继续由发布领域创建，不在 adapter 内复制发布事实源 |
| 外部动作 | AC-5 不负责抖音扫码/授权/上传/最终发布；只产出本地视频/封面/发布素材并交给发布中心 |
| 回滚 | 新 flag 关闭后旧 StudioApp 口播、session 恢复、资产和发布路径保持可用；新 adapter 失败不能删除旧 session/旧产物 |

## 冻结映射表（Entry 契约）

### Session / Task / AppRun

| 既有 session 事实 | Generic Task 投影 | AppRun 语义 | 说明 |
| --- | --- | --- | --- |
| 创建/输入未就绪；step status `pending`/`ready` | `pending` | `draft`/`queued` | 只记录稳定 `session_id` 与 `app_run_id`，不复制 session 全量 state |
| 单步执行中；step status `running` | `running` | `running` | `task_type=ip_broadcast_step`，`step_key` 指向既有六步之一 |
| 需要用户编辑、确认选题/文案/素材/数字人 | `needs_review` 或 `needs_attention` | `needs_review` | 不得投影为 completed；保留当前 session step 与 notice |
| 旧 workflow/provider 可重试错误 | `failed` | `failed` | error code/diagnostic 脱敏；重试创建新 attempt，不删除旧 session/产物 |
| 用户取消或应用关闭后安全停手 | `cancelled` | `cancelled` | 取消必须幂等，不能把已完成 ArtifactVersion 标为失败 |
| 六步完成且 final video/cover/publish copy 已登记 | `completed` | `completed` | 只在 artifact 登记与文件可信校验完成后允许 |

### 来源与产物

| 来源入口 | Adapter 输入 | 固定来源 |
| --- | --- | --- |
| 空白项目 | `project_id` + 用户输入快照 | 无外部 ArtifactVersion；session 新建后由 workflow 生成 |
| 文案 | `source_artifact_version_ids=[copywriting@vN]` | project、artifact、version 三者校验通过后写入 session context |
| 标题 | `source_artifact_version_ids=[selected_title@vN]` | 同上；标题只作为发布标题/脚本上下文，不覆盖既有文案事实 |

最终视频、封面、发布文案分别登记为 `video`、`cover`、`publish_copy` ArtifactVersion；文件只能来自受信资产/渲染根目录，PublishPackage 仍由发布领域从这些版本固化。

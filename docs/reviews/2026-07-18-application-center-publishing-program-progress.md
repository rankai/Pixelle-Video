## 1. 使用规则

1. 任何时刻只能有一个 Stage 为 `in_progress` 或 `waiting_user`；
2. 上一 Program Gate 未 `passed`，下一 Stage 不得开始；
3. 每次开始/结束工作都更新本文件；
4. 测试未运行写 `not_run`，不得留空或写“应当通过”；
5. 阻塞必须写明证据、影响、恢复条件和负责人；
6. 顺序或共同契约变化必须链接 Change Request；
7. 进度百分比只按通过的 Program Gate 计算，不按代码行或主观完成度计算。
8. 历史工作日志保留当时的下一步与边界；当前入口只看本文头部状态、当前 Stage 控制卡和最新工作日志。

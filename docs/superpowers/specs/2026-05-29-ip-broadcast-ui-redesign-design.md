# IP 口播桌面端 UI 改造设计方案

## Summary

本方案用于改造 React/Tauri 版老板 IP 口播桌面端的视觉与信息架构。当前页面功能已基本可用，但视觉偏“纯白表单堆叠”，ToB 产品感不足，发布页信息混杂。改造目标是让产品更像一个可长期使用的业务生产工作台，而不是临时工具页。

最终方向：

- 默认视觉方向：专业工作台风。
- 默认主题：石墨蓝专业版。
- 可选皮肤：运营温度版、品牌表现版。
- 皮肤入口：配置页的外观设置。
- 发布页重构为“发布素材包交付台”。
- 品牌包与界面皮肤分离：皮肤控制 App UI，品牌包控制视频模板、封面、发布素材和业务信息。

## Product Positioning

目标用户是中小企业老板、门店运营、代运营团队，用于快速生成口播、探店、本地生活短视频。界面需要满足：

- 可信：看起来像正式生产系统，不像 demo。
- 高效：用户知道当前在第几步、缺什么、下一步做什么。
- 可控：资产、任务、配置、发布素材各自有清晰入口。
- 业务化：表达使用门店、品牌包、套餐、优惠、平台发布这些业务概念。
- 克制：不做大面积渐变、营销页式 hero、过度装饰和低密度大卡片。

## Design References

参考主流 ToB / 生产力产品的共同设计规律，不直接照搬：

- Ant Design：后台布局、8px 间距体系、固定导航和动态工作区。
- Shopify Polaris：商家后台的一致体验和业务动作导向。
- Microsoft Fluent 2：design tokens、状态色、可访问对比。
- Linear：低噪音界面、聚焦当前任务、克制的层级色。

落到本项目的设计原则：

- 低饱和背景承载长时间工作。
- 深色侧栏建立产品识别。
- 主色只用于主要动作、当前步骤和焦点状态。
- 状态色只表达状态，不做装饰。
- 卡片数量可控，卡片内信息密度提高。

## Visual System

### Default Theme: Graphite Blue

默认主题为“石墨蓝专业版”。

建议 token：

- `--color-bg`: `#F3F6FA`
- `--color-surface`: `#FFFFFF`
- `--color-sidebar`: `#172033`
- `--color-sidebar-muted`: `#263449`
- `--color-primary`: `#2F6FED`
- `--color-primary-soft`: `#DBEAFE`
- `--color-text`: `#111827`
- `--color-text-muted`: `#64748B`
- `--color-border`: `#DCE4EE`
- `--color-success`: `#16A34A`
- `--color-success-soft`: `#DCFCE7`
- `--color-warning`: `#D97706`
- `--color-warning-soft`: `#FEF3C7`
- `--color-danger`: `#DC2626`
- `--color-danger-soft`: `#FEE2E2`

建议尺寸：

- 页面外边距：24px。
- 卡片圆角：12px 到 14px。
- 输入框圆角：8px。
- 主按钮高度：38px 到 42px。
- 卡片间距：12px 到 16px。
- 页面主栅格：左侧导航 220px，右侧工作区自适应。

### Optional Skins

保留三套皮肤：

1. 石墨蓝专业版
   - 默认启用。
   - 适合长期使用和通用 ToB 场景。

2. 运营温度版
   - 主色偏暖，例如 `#D97706`。
   - 适合门店运营用户，但暖色面积必须克制。

3. 品牌表现版
   - 主色可用 `#C2410C` 作为默认强调。
   - 后续可作为品牌包换肤的基础，但首版不让品牌包直接覆盖系统 UI。

皮肤切换只影响 App Shell、按钮、状态、卡片和导航，不改变视频模板生成逻辑。

## Information Architecture

桌面端采用固定左侧导航：

- IP 口播工作台
- 素材资产
- 任务中心
- 配置诊断

右侧工作区根据当前菜单变化。IP 口播工作台结构：

1. 顶部生产主控台
2. 1-6 横向步骤条
3. 单步工作区
4. 底部动作栏

不再使用全局三列卡片堆叠作为主形态。三列布局适合早期测试，但对 6 步生产流程和发布交付不够清晰。

## IP Broadcast Workspace

### Production Console

生产主控台需要显示：

- 当前品牌包和业务预设。
- 当前进度，例如 `5/6 已完成`。
- 下一步动作。
- 缺失项提示。
- 当前任务状态和停止按钮。

主控台只保留一个强主按钮：

- 无任务时：`一键继续：下一步动作名称`
- 任务运行中：显示进度信息和停止按钮。
- 任务失败时：显示业务化错误原因和重试入口。

### Step Bar

步骤条横向展示 6 步：

1. 素材来源
2. 文案确认
3. 声音生成
4. 数字人视频
5. 一键成片
6. 视频发布

状态文案统一业务化：

- 未开始
- 可执行
- 进行中
- 已完成
- 失败

不在 UI 中暴露 `pending`、`ready`、`done` 等工程状态词。

### Single Step Workspace

每次只聚焦一个步骤。步骤内部布局采用：

- 左侧或上方：主要配置。
- 右侧或下方：摘要、预览、资产选择结果。
- 底部：当前步骤主操作。

原则：

- 主按钮放在配置完成后的区域。
- 高级设置默认收起。
- 资产选择用统一卡片组件。
- 路径、workflow id、文件名等技术信息默认不作为主信息。

## Publish Package Delivery Page

第 6 步重构为“发布素材包交付台”，解决当前发布页字段堆叠、布局混乱的问题。

布局：

- 左侧主区域：发布信息。
- 右侧固定区域：视频预览、封面、文件信息、下载动作。

顶部摘要卡：

- 状态：发布素材包已准备好。
- 主动作：
  - 下载最终视频
  - 复制全部
  - 下载 JSON

通用发布信息：

- 标题
- 描述
- 标签
- 口播文案

每个字段提供独立复制按钮。长文本区域保持可读，不让按钮和文本混在同一行。

平台建议区：

- 抖音版
- 小红书版
- 视频号版
- 快手版

每个平台卡片包含：

- 平台名
- 推荐标题
- 推荐描述
- 推荐标签
- 一键复制该平台素材

右侧视频信息：

- 视频预览或封面预览。
- 文件路径摘要。
- 时长、分辨率、文件大小、生成时间。
- 下载视频按钮。

## Theme Settings

皮肤入口放在配置页：

- 菜单：配置诊断。
- 分组：外观设置。
- 控件：皮肤选择卡片或 segmented control。
- 默认值：石墨蓝专业版。

保存策略：

- 前端保存到 localStorage，保证用户下次打开仍使用同一皮肤。
- 后续桌面端配置稳定后，可同步写入 desktop config。

首版不在工作台右上角放皮肤切换，避免影响生产流程。

## Asset Pages

素材资产页统一视觉规格：

- 音色库
- 形象库
- 画面模板库
- 视频素材库
- 品牌包

统一卡片结构：

- 缩略图或类型图标。
- 名称。
- 简短描述。
- 状态或标签。
- 主操作。
- 次级操作。

资产页需要比工作台更适合维护和浏览：

- 卡片高度统一。
- 文案最多两行，完整信息用 title 或展开详情。
- 删除、更新、选择等操作区固定在卡片底部。

## Error Handling And State Feedback

错误提示必须业务化：

- 不直接展示底层异常作为唯一文案。
- 保留技术详情入口，供排查使用。

示例：

- `数字人生成失败：RunningHub 鉴权失败，请检查 API Key。`
- `语音生成失败：参考音频不可用，请重新选择音色或上传新的参考音频。`
- `成片失败：最终视频合成失败，请检查 BGM 路径和覆盖视频素材。`

长任务状态：

- 主控台展示任务阶段。
- 任务中心展示历史任务。
- 当前步骤底部展示本步骤结果。
- 停止按钮只停止本地等待；如果远程任务无法取消，需要明确提示。

## Implementation Phases

### Phase 1: Design Token And Theme System

目标：

- 建立 CSS token。
- 实现三套皮肤。
- 配置页增加外观设置。

验收标准：

- 默认主题为石墨蓝。
- 配置页可切换三套皮肤。
- 刷新后保留用户选择。
- 不影响现有 API 和生产流程。

### Phase 2: App Shell Redesign

目标：

- 改为左侧深色导航 + 右侧工作区。
- 统一顶部主控台和步骤条视觉。

验收标准：

- 1280px 宽度下无拥挤和横向滚动。
- 导航、主控台、步骤条层级清楚。
- 当前任务状态可读。

### Phase 3: Workspace Component Polish

目标：

- 统一步骤卡片、资产卡片、按钮、notice、empty state。
- 工程状态词改为业务文案。

验收标准：

- 1-6 步按钮位置一致。
- 每步末尾只有一个主要状态提示。
- 资产卡片高度统一。

### Phase 4: Publish Package Page

目标：

- 重构第 6 步为发布素材包交付台。
- 平台建议区独立成卡片。

验收标准：

- 下载视频、复制标题、复制描述、复制标签、复制平台素材路径清晰。
- 发布页不再出现字段堆叠混乱。
- 文案长时不撑乱布局。

### Phase 5: Browser Regression

目标：

- 验证关键视口和核心流程。

验收标准：

- 1280x800、1440x900、1920x1080 页面可用。
- 走通至少一次火锅店老板场景。
- `uv run ruff check .` 通过。
- `uv run pytest -q` 通过。
- `npm run build` 通过。

## Out Of Scope

本轮不做：

- 真实发布平台 API。
- 拖拽式时间轴。
- 品牌包直接覆盖系统 UI。
- 全量 Home 页面迁移。
- 移动端适配。

## Open Decisions

已确认：

- 默认采用石墨蓝专业版。
- 运营温度版、品牌表现版作为可选皮肤。
- 皮肤入口放在配置页。
- 发布页重构为发布素材包交付台。

未进入本轮：

- 是否允许企业团队管理员锁定统一皮肤。
- 是否支持品牌包影响 App UI 强调色。
- 是否给不同业务预设提供专属模板主题。
